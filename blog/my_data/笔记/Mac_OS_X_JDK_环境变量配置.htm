我的Mac系统版本是OS X 10.10.2 Yosemite，为了用Eclipse做android开发，安装了jdk 1.7, 但是如果想使用IntelliJ IDE做android开发的话，就需要安装jdk1.6,于是按照官方指引安装了jdk1.6，但是运行java -version命令时，现实的系统jdk版本还是1.7，于是在网上找了半天，终于找到了修改系统默认jdk版本的办法，

方法1:进入命令行

cd ~

touch .bash_profile

vi .bash_profile

PS: 如果home目录下没有.bash_profile文件，就自己新建一个

输入内容jdk变量配置内容：

export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.7.0_xx.jdk/Contents/Home #jdk安装路径   
export PATH=$JAVA_HOME/bin:$PATH 
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar

输入：wq保存。

更新配置文件：

source .bash_profile

PS:这里的 JAVA_HOME路径是自己从oracle官网上下载的安装包jdk1.7的安装路径，而从官网 https://support.apple.com/kb/DL1572?locale=en_US  上下载的jdk1.6默认的安装路径是 /System/Library/Java/JavaVirtualMachines/1.6.0.jdk/Contents/Home, 按上面的方法，把JAVA_HOME改成jdk1.6的路径，就可以办系统默认jdk改成1.6了。

但是我又发现，想使用IntelliJ IDE做android开发，其实不用修改系统默认的jdk版本，只需要在IntelliJ IDE中导入工程时，选择jdk1.6的安装路径，就可以使用不同版本的jdk了,希望可以帮到有我刚开始的困惑的人。

方法2: 修改文件链接

OS X 自带JDK

这个JDK由APPLE维护，也是OSX默认的JDK，它的路径是：

/System/Library/Frameworks/JavaVM.framework/Versions/CurrentJDK/Home

这其实是一个符号链接，它指向：

/System/Library/Java/JavaVirtualMachines/1.6.0.jdk/Contents/Home

OS X自带的JRE

/System/Library/Frameworks/JavaVM.framework/Versions/Current

ORACLE的JDK7

/Library/Java/JavaVirtualMachines/jdk1.7.0_xx.jdk/Contents/Home

ORACLE的JRE7

/Library/Internet Plug-Ins/JavaAppletPlugin.plugin/Contents/Home

OPENJDK7

/Library/Java/JavaVirtualMachines/jdk1.7.0.jdk/Contents/Home

修改方法: 打开终端,输入:

sudo unlink /System/Library/Frameworks/JavaVM.framework/Versions/CurrentJDK

sudo ln -s /Library/Java/JavaVirtualMachines/jdk1.7.0_xx.jdk/Contents /System/Library/Frameworks/JavaVM.framework/Versions/CurrentJDK
