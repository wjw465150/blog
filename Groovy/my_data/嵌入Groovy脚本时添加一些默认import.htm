嵌入Groovy脚本时添加一些默认import

刚才给在做的项目里加了些便捷功能，把做法记下来。 
我们在Java应用里嵌入了Groovy脚本。为了方便，业务里经常用的一些类可以默认import进来，让大家少写点重复代码。幸好，这种功能用Groovy来做毫不费力。 

Java代码
 
  1. import groovy.lang.GroovyClassLoader;  
  2.   
  3. import java.security.CodeSource;  
  4.   
  5. import org.codehaus.groovy.ast.ClassHelper;  
  6. import org.codehaus.groovy.ast.ModuleNode;  
  7. import org.codehaus.groovy.control.CompilationFailedException;  
  8. import org.codehaus.groovy.control.CompilationUnit;  
  9. import org.codehaus.groovy.control.CompilerConfiguration;  
  10. import org.codehaus.groovy.control.Phases;  
  11. import org.codehaus.groovy.control.SourceUnit;  
  12. import org.codehaus.groovy.control.CompilationUnit.SourceUnitOperation;  
  13.   
  14. public class CustomGroovyClassLoader extends GroovyClassLoader {  
  15.     private final String[] DEFAULT_IMPORT_CLASSES = {  
  16.         "rednaxelafx.sample.Foo",  
  17.         "rednaxelafx.sample.Bar"  
  18.     };  
  19.       
  20.     /** 
  21.      * Validate classes. 
  22.      * 
  23.      * Because the specified default import classes might not be present in 
  24.      * the during runtime, they have to be validated first, otherwise Groovy 
  25.      * won't compile the code. 
  26.      *  
  27.      * @param className the name of the class to validate 
  28.      * @return true iff the class exists and is valid 
  29.      */  
  30.     private boolean isClassValid(String className) {  
  31.         try {  
  32.             loadClass(className, false);  
  33.             return true;  
  34.         } catch (ClassNotFoundException e) {  
  35.             return false;  
  36.         }  
  37.     }  
  38.       
  39.     private static String getClassSimpleName(final String className) {  
  40.         return className.substring(className.lastIndexOf('.') + 1);  
  41.     }  
  42.       
  43.     private static boolean alreadyImported(final String alias, final ModuleNode ast) {  
  44.         return ast.getImport(alias) != null;  
  45.     }  
  46.       
  47.     /** 
  48.      * add default imports 
  49.      */  
  50.     @Override  
  51.     protected CompilationUnit createCompilationUnit(  
  52.             CompilerConfiguration config, CodeSource source) {  
  53.         CompilationUnit compUnit = super.createCompilationUnit(config, source);  
  54.         compUnit.addPhaseOperation(new SourceUnitOperation() {  
  55.             public void call(SourceUnit source) throws CompilationFailedException {  
  56.                 ModuleNode ast = source.getAST();  
  57.                 for (String className : DEFAULT_IMPORT_CLASSES) {  
  58.                     String simpleClassName = getClassSimpleName(className);  
  59.                     if (isClassValid(className) && !alreadyImported(simpleClassName, ast)) {  
  60.                         ast.addImport(simpleClassName, ClassHelper.make(className));  
  61.                     }  
  62.                 }  
  63.             }  
  64.         }, Phases.CONVERSION);  
  65.         return compUnit;  
  66.     }  
  67. }  


直接用ast.addImport()的话，这些添加的“默认”import实际上是被append到原本的import列表的后面了。Groovy里后import的比先import的要更优先，所以这种默认import可能会带来干扰。要注意。
