JAVA中基本数据类型中short整数和char整数二者的区别

java中short类型赋值给char类型，结果输出问号（?）的原因。

我们通过ASCII码可以知道，字符实质上也是整数，那么有下面这样一个问题，在java中会存在。
short整数是2个字节，char整数也是2个字节，那么二者的区别是什么?
在JAVA中，short类型的取值范围是-32768~32767，是有符号数，首位为符号位，首位为1时，此整数位负数。
char类型的取值范围是0~65535 ，是无符号数，首位无论是0或1，代表的都是正数。

下面看一个例子：
char c1 = -1;//这是个错误的写法。
但是，
char c2 = 65;//是正确的。
这是因为char类型的取值范围不包括负数。
然而：
short s2 = -1；
char c3 = s2;//short类型的s2是负数，将s2赋值给char，也是不可以的，因为虽然short的取值范围中包括负数，但是编译器无法保证s2中一定是个正整数，编译时会报检查的错误。
System.out.println(s3);//此结果是：？ 因为如果char中存储的是负数，则无法根据ASCII、Unicode找到对应字符，进而JAVA使用“”？“”代表。
