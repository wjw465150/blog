之前有很多前辈写过我们经常犯的一些低级错误：

一个隐形的java int溢出

竟然犯下这种低级错误，羞愧难当啊！！！

 

在此我再补充一个，好时刻提醒自我保持清醒！ 这个是关于final变量的，有时候我们可能定义一些常量类 这里边存放了我们定义的一些常量，有时候我们可能修改，修改后有时候为了简单 仅仅把这个类编译后 部署到服务器，此时我们可能看到的还是老的数据。这是因为final变量的编译期优化造成的，所有使用final变量的位置会通过值副本替换 而不是引用。测试场景Hotspot1.6

 

Java代码
 
  1. public class A {  
  2.     //网站的管理员ID  
  3.     public static final int ADMIN_ID = 10000001;  
  4. }  

 

 

Java代码
 
  1. public class B {  
  2.     public static void main(String[] args) {  
  3.         int adminId = A.ADMIN_ID;  
  4.         System.out.println(adminId);  
  5.     }  
  6. }  

 

 

此时通过

javac *.java 编译所有类；

java B 运行结果是没有问题的。

 

接下来A的ADMIN_ID变了，我修改为10000002

 

Java代码
 
  1. public static final int ADMIN_ID = 10000002;  

 

javac A.java 编译A类；

Java B  运行结果还是10000001。

此时我们通过javap -verbose B 查看B的字节码

 

Java代码
 
  1. public static void main(java.lang.String[]);  
  2.   Code:  
  3.    Stack=2, Locals=2, Args_size=1  
  4.    0:   ldc     #2; //int 10000001  
  5.    2:   istore_1  
  6.    3:   getstatic       #3; //Field java/lang/System.out:Ljava/io/PrintStream;  
  7.    6:   iload_1  
  8.    7:   invokevirtual   #4; //Method java/io/PrintStream.println:(I)V  
  9.    10:  return  

 此时我们看到ldc #2;//int 10000001 没有变，到此我们知道原因了，这是编译的优化造成的。

 

怎么解决呢？

1、把所有相关类编译部署；

2、通过如下方案解决：

 

Java代码
 
  1. public class A {  
  2.     //网站的管理员ID  
  3.     public static final int ADMIN_ID;   
  4.     static {  
  5.         ADMIN_ID = 10000003;   
  6.     }  
  7. }  

 

 此时ADMIN_ID的初始化 放在static初始化块中，编译期不会发生优化。再查看B类字节码，发现是通过getstatic获取常量的：

 

Java代码
 
  1. public static void main(java.lang.String[]);  
  2.   Code:  
  3.    Stack=2, Locals=2, Args_size=1  
  4.    0:   getstatic       #2; //Field A.ADMIN_ID:I  
  5.    3:   istore_1  
  6.    4:   getstatic       #3; //Field java/lang/System.out:Ljava/io/PrintStream;  
  7.    7:   iload_1  
  8.    8:   invokevirtual   #4; //Method java/io/PrintStream.println:(I)V  
  9.    11:  return  

 

同理 对应final实例变量/局部变量也是有这个问题的：

Java代码
 
  1. public class C {  
  2.     //网站的管理员ID  
  3.     public final int ADMIN_ID = 10000001;  
  4. }  

 

 

Java代码
 
  1. public class D {  
  2.     public static void main(String[] args) {  
  3.         int adminId = new C().ADMIN_ID;  
  4.         System.out.println(adminId);  
  5.     }  
  6. }  

 

我们仅需要将C改为：

 

Java代码
 
  1. public class C {  
  2.     //网站的管理员ID  
  3.     public final int ADMIN_ID;   
  4.     {  
  5.         ADMIN_ID = 10000001;  
  6.     }  
  7. }  

 或

 

Java代码
 
  1. public class C {  
  2.     //网站的管理员ID  
  3.     public final int ADMIN_ID;   
  4.     public C() {  
  5.         ADMIN_ID = 10000001;  
  6.     }  
  7. }
